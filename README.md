# 终极面试官 1.0 Skill [edison9l 团队版]

## 一句话定位

一个让 AI 从"随机问面试题"升级为"能读简历、控节奏、追问核验、处理候选人反问、生成交接档案"的面试官 Skill。

---

## 核心模块总览

| 模块 | 核心功能 | 解决的问题 | 亮点 |
|------|---------|----------|------|
| **1. 自然启动模块** | 用户输入"开始面试"后自然开场 | 避免一上来甩表格、问后台问题 | 候选人端像真人面试，不像机器人 |
| **2. 管理员配置模块** | 企业提前配置公司、岗位、薪资、福利 | AI 不知道公司信息时乱答 | 有配置优先用，无配置则记录待 HR 回复 |
| **3. 材料接入模块** | 支持 PDF、图片、文字简历、JD、作品链接 | 候选人材料格式混乱 | 长简历可摘要，多页 PDF 可分层提取 |
| **4. 简历时间线审计** | 检查年龄、学历、毕业时间、空窗期、跳槽 | AI 只总结简历，不会发现逻辑风险 | 能筛出 19 岁毕业两年、3 年 5 份工作等异常 |
| **5. 候选人声明账本** | 记录"带过 50 人""10 万播放"等关键声明 | AI 无法验证真假 | 不判断真假，而是生成追问、补材料、背调问题 |
| **6. 场景包机制** | 按岗位加载不同面试维度 | 所有岗位用同一套题 | 支持新媒体、销售、技术、管理、转岗、校招等 |
| **7. 面试轮次模块** | 区分 HR 初面、业务二面、老板终面 | 三轮面试问法混乱 | 每轮目标、问题、输出、淘汰标准不同 |
| **8. 微轮追问模块** | 每次只问一个主问题，按回答追问 | 一口气问太多，像审讯 | 对话更像真人面试官 |
| **9. 回答诊断模块** | 判断答非所问、空泛、无证据、逻辑断裂 | AI 不知道用户答得好不好 | 根据回答质量决定追问、跳题、升压或收口 |
| **10. 候选人反问路由** | 处理薪资、双休、公司、岗位等问题 | 候选人一问，AI 就出戏 | 能答则答，不能答就记录待 HR 回复 |
| **11. 节奏控制模块** | 控制题量、时长、追问深度 | 面试过长，候选人没耐心 | 支持最后两问、快速收口、礼貌打断 |
| **12. 流程交接模块** | 每轮生成上下文胶囊 | 二面三面丢失上一轮信息 | 下一轮知道已问什么、还要问什么 |
| **13. 候选人档案模块** | 生成可保存的候选人档案 | 多轮面试无法追踪 | 可保存、加载、判断初面/二面/三面/淘汰 |
| **14. 内外报告分离** | 候选人看外部话术，管理员看内部报告 | 把内部红旗直接发给候选人不专业 | 候选人端克制，管理员端完整 |
| **15. 合规边界模块** | 避免婚育、民族、宗教等无关敏感问题 | 面试容易踩合规风险 | 只问岗位相关内容 |
| **16. 统一输出模块** | 输出《出手前闸门报告｜终极面试官》 | 报告风格不统一 | 可和"验明/暗礁"等 Skill 组成套件 |

## 项目核心卖点

| 卖点 | 解释 |
|------|------|
| **不像题库** | 根据岗位、简历、回答动态问，不是套模板 |
| **不像表单** | 候选人端自然开场，不甩配置表 |
| **不装懂** | 不知道就记录待 HR 回复，不乱编 |
| **不乱判真假** | 用声明账本做核验，不直接否定 |
| **不拖时间** | 有题量和节奏控制 |
| **不丢上下文** | 有候选人档案和面试胶囊做多轮衔接 |
| **可适配不同岗位** | 靠场景包和管理员维度配置 |
| **可企业部署** | 可配置公司信息、岗位、薪资、福利 |
| **支持流程面试** | HR 初面、业务二面、老板终面真正衔接 |
| **有比赛记忆点** | "不是问面试题，而是让 AI 像面试官一样控场" |

## 最简版总结

**终极面试官 1.0 Skill [edison9l 团队版] 的优势是：**

它不是一个面试题库，而是一套让 AI 能 **读简历、识别风险、动态追问、处理反问、控制节奏、生成档案、衔接多轮面试** 的 Skill。

**它解决的是普通 AI 面试工具最弱的地方：**
- ❌ 不会看简历逻辑 → ✅ 简历时间线审计
- ❌ 不会追问核验 → ✅ 候选人声明账本
- ❌ 不会处理候选人反问 → ✅ 反问路由模块
- ❌ 不会控制节奏 → ✅ 节奏控制模块
- ❌ 不会区分 HR/业务/老板 → ✅ 面试轮次模块
- ❌ 不会生成可交接的面试记录 → ✅ 候选人档案 + 上下文胶囊
